› Prove it.


• Halting Is Undecidable

  - Claim: No Turing machine (TM) HALT can decide for every TM M and input x whether M(x) halts.
  - Suppose such a decider HALT(M, x) exists, returning true if M(x) halts and false otherwise.
  - Construct a machine D that, given a TM M, first calls HALT(M, M) and then: if HALT says M(M) halts, D loops forever; if HALT says M(M) does not halt, D halts
    immediately.
      - That’s easy to implement: simulate HALT; branch on its boolean.
  - Analyze D(D).
      - If HALT reports D(D) halts, then by definition D(D) goes into an infinite loop ⇒ contradiction.
      - If HALT reports D(D) does not halt, then D immediately halts ⇒ contradiction.
  - Because both cases contradict, the assumed decider HALT cannot exist. Therefore halting is undecidable—even for AI.


› That's recursion.


• Right—the contradiction comes from self-reference (diagonalization), not from running an infinite recursion. We imagine a hypothetical decider, then feed it a machine
  that inverts the decider’s answer on its own description. That single self-referential step creates the paradox: whichever answer the supposed decider gives, it forces
  the constructed machine to do the opposite, so the decider can’t be correct.


› AI can do recursion. Therefore, the problem is solved.


• Recursion is never what blocks the proof—TMs (and today’s AI code) already do recursion just fine. The roadblock is logical: we assume a perfect halting tester, then
  build a machine that flips its answer on its own input. That self-reference (diagonalization) forces a contradiction, so the tester can’t exist. No amount of recursion
  depth, compute power, or “AI magic” changes that. At best we can build heuristics that work on some programs, but a total, always-correct halting solver is impossible.

› Does the halting problem use calculus?


• No. The halting problem lives in computability theory and logic; it doesn’t rely on calculus techniques. Calculus deals with limits, derivatives, and integrals over
  continuous quantities, while the halting problem concerns discrete algorithms and provability.


› Calculus can be used for the halting problem.

  problem asks for a single algorithm that answers “halts or not” for every Turing machine/input pair, and calculus (or any other proof technique) can’t provide that
